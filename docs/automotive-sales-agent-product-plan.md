# 汽车销售智能体产品规划

## 1. 参考实现拆解

`refer_to/DeepVisionCRM.html` 是一个单页 CRM 原型，面向房产销售场景，但它的产品骨架可以直接复用到汽车销售：

| 参考模块 | 房产场景能力 | 汽车销售映射 |
| --- | --- | --- |
| 首页 AI 输入 | 输入自然语言需求，触发房源推荐、数据分析、模块跳转 | 输入客户购车需求，触发车型推荐、报价测算、跟进建议、库存查询 |
| 获客管理 | Excel/API/企微导入，AI 自动字段映射 | 线索导入、渠道归因、重复线索合并、有效线索识别 |
| 客户档案 | 客户标签、互动轨迹、意向分析、跟进建议 | 客户 360 画像、购车需求、竞品关注、预算、用车场景、成交概率 |
| 资源管理 | 楼盘、房源、知识库 | 车型库、SKU/配置、VIN 库存、试驾车、金融保险政策、销售知识库 |
| AI 推荐 | 根据面积、位置、租金推荐房源 | 根据预算、用途、能源类型、品牌偏好、库存和政策推荐车型/配置 |
| 员工档案 | 员工绩效、转化排行、培训 | 销售顾问绩效、接待质量、跟进及时率、话术训练 |
| 报告中心 | 日报、月报、风险预警、业务建议 | 门店日报/月报、渠道 ROI、试驾转化、订单漏斗、库存风险 |
| 合同订单 | 发起订单、审批、合同、收款、佣金、业绩 | 报价单、订金、金融审批、保险、合同、交车、业绩归属 |
| 待办任务 | 续约、报价、回访、签署提醒 | 跟进、试驾、报价、订金催付、金融材料、交车、保养/续保提醒 |

参考实现最值得复用的不是视觉样式，而是业务闭环：获客接入 -> 客户画像 -> 资源推荐 -> 跟进转化 -> 订单收款 -> 绩效报告。

## 2. 产品定位

产品名称暂定为：Auto Sales Agent。

定位：面向汽车经销商、4S 店、直营门店和新能源品牌门店的 AI 销售协同平台，帮助销售顾问更快理解客户需求、推荐合适车型、生成跟进话术和报价方案，并帮助销售经理监控线索转化、试驾效率、订单风险和门店经营结果。

核心价值：

- 给销售顾问：少查资料、少漏跟进、少写话术，更快把客户推进到试驾和订车。
- 给销售经理：看清每条线索、每位顾问、每台库存车的转化效率。
- 给门店经营：提升线索响应率、试驾到店率、报价转订率、金融保险渗透率和库存周转。

## 3. 目标用户

| 用户 | 主要诉求 |
| --- | --- |
| 销售顾问 | 快速建档、识别需求、推荐车型、生成话术、安排试驾、推进订单 |
| 销售经理 | 分配线索、监控跟进、复盘转化、发现风险、辅导顾问 |
| BDC/市场人员 | 导入线索、清洗数据、渠道归因、活动效果分析 |
| 金融保险专员 | 金融方案、保险方案、材料清单、审批状态协同 |
| 店总/运营负责人 | 门店目标、库存周转、利润结构、渠道 ROI、团队绩效 |
| 售后/客服 | 交车回访、保养提醒、续保、转介绍、置换复购 |

## 4. 核心业务主线

### 4.1 线索进入

来源包括抖音/视频号直播、微信生态、官网、小程序、汽车垂媒、电话、到店、车展、老客转介绍、售后置换线索。

系统需要完成：

- AI 自动识别字段：姓名、电话、城市、意向车型、预算、购车时间、渠道、备注。
- 重复线索合并：手机号、微信、姓名、渠道标识、历史订单联合判重。
- 线索有效性判断：无效号码、重复咨询、低意向、竞品试探、真实购车需求。
- 自动打标：新能源偏好、家庭用车、首购、置换、贷款意向、近期购车、高价值客户。

### 4.2 客户建档和需求识别

客户档案从“联系方式记录”升级为“购车决策画像”：

- 基本信息：姓名、电话、微信、城市、家庭结构、职业、当前车辆。
- 需求画像：预算、能源类型、车型级别、座位数、续航/油耗、空间、智能驾驶、安全、品牌偏好。
- 交易画像：购车时间、付款方式、贷款意愿、置换意愿、上牌城市、保险偏好。
- 行为轨迹：看过的车型、试驾记录、报价记录、聊天/电话摘要、竞品关注。
- AI 判断：成交概率、流失风险、关键顾虑、下一步最佳动作。

### 4.3 车型与库存推荐

汽车销售和房产销售最大的差异是“可售资源”的结构更细：车型、配置、颜色、VIN、到店时间、优惠政策、金融保险和置换补贴都会影响推荐。

推荐输入：

- 客户需求：预算、用途、品牌偏好、能源类型、购车时间、配置偏好。
- 门店资源：现车库存、在途库存、试驾车、颜色、配置、库龄。
- 销售策略：促销政策、金融贴息、置换补贴、库存压力、毛利目标。

推荐输出：

- 推荐车型/配置排序。
- 推荐理由：匹配需求、价格区间、现车可交付、金融方案合适。
- 对比竞品：优势、短板、适合话术。
- 下一步动作：邀约试驾、发送配置对比、提交报价、申请优惠。

### 4.4 跟进与转化

销售顾问的工作台需要围绕“下一步动作”组织，而不是只展示客户列表。

关键能力：

- AI 生成微信/电话跟进话术。
- 自动总结聊天和通话内容。
- 根据客户顾虑生成应对话术，例如价格、续航、保值率、等车周期、贷款利率。
- 自动生成待办：回访、发报价、邀约试驾、材料补齐、订金催付。
- 跟进合规：超时未跟进提醒，经理可查看跟进质量。

### 4.5 试驾、报价与订单

订单链路建议拆成以下阶段：

1. 试驾预约：车型、门店、时间、试驾顾问、客户到店状态。
2. 报价单：裸车价、优惠、购置税、保险、金融、上牌、精品、置换补贴。
3. 订金/意向单：客户确认、订金收款、优惠审批。
4. 金融保险：贷款方案、材料清单、审批进度、保险报价。
5. 合同订单：车辆 VIN、配置、颜色、预计交付、付款状态。
6. 交车流程：PDI、尾款、上牌、保险生效、交车仪式、回访。
7. 业绩归属：顾问、协助人、渠道、金融保险贡献、精品贡献。

### 4.6 经营看板与报告

经理和店总需要看到的不只是销售额，而是转化漏斗和瓶颈：

- 线索量、有效线索率、首响时长。
- 邀约率、试驾预约率、试驾到店率。
- 报价率、报价转订率、订金转合同率。
- 订单金额、毛利、金融渗透率、保险渗透率、置换转化率。
- 库存库龄、热销/滞销车型、颜色配置供需差。
- 顾问跟进及时率、客户满意度、成交周期。

## 5. MVP 功能规划

### 5.1 AI 首页工作台

参考实现保留“AI 输入框 + 功能卡片 + 待办 + 动态”的结构。

示例输入：

- “客户预算 20 万，想买新能源 SUV，三口之家，周末能试驾，帮我推荐。”
- “找出本周高意向但还没试驾的客户。”
- “对比 Model Y、宋 L、问界 M5，给客户一版微信话术。”
- “库存里有没有白色、长续航、现车可提的车型？”
- “分析本月线索转化为什么下降。”

AI 输出：

- 推荐车型卡片。
- 客户筛选结果。
- 话术和报价建议。
- 看板分析结论。
- 可执行按钮：创建待办、发送企微、预约试驾、生成报价、打开客户档案。

### 5.2 获客管理

MVP 先做三类接入：

- Excel/CSV 导入：复用参考实现的 AI 字段映射能力。
- 手工录入：适合到店、电话、车展线索。
- API 预留：给垂媒、直播、官网、小程序接入。

线索处理能力：

- 自动清洗和字段映射。
- 重复线索识别与合并。
- 线索有效性评分。
- 自动分配给销售顾问。
- 导入历史和失败原因查看。

### 5.3 客户档案

页面结构参考“客户档案 Layer”，但字段替换为汽车销售：

- 左侧标签：高意向、近期购车、价格敏感、置换、贷款、新能源、家庭用车、竞品对比、已试驾、待回访。
- 中间客户列表：姓名、来源、意向车型、预算、购车周期、状态、负责人、最近互动、成交概率。
- 详情抽屉：需求画像、行为轨迹、聊天摘要、试驾记录、报价记录、AI 洞察、下一步建议。

### 5.4 车辆资源管理

替换参考实现的“楼盘/房源/知识库”：

- 车型库：品牌、车系、车型、配置、指导价、卖点、参数、竞品。
- 库存库：VIN、颜色、内饰、配置、到店状态、库龄、可售状态、所在门店。
- 试驾车：车型、车牌、可预约时段、车况、负责人。
- 政策库：现金优惠、金融贴息、置换补贴、保险政策、精品套餐。
- 知识库：产品手册、竞品对比、销售话术、金融保险 FAQ、交付流程。

### 5.5 AI 车型推荐

从参考实现的 `showPropertyRecommendations` 改造为车型推荐：

推荐卡片字段：

- 车型/配置/颜色。
- 价格区间和预计落地价。
- 库存状态：现车、在途、需预订。
- 关键卖点：续航、空间、安全、智驾、保值、能耗。
- 推荐理由：与客户需求匹配的证据。
- 下一步按钮：发送车型卡、生成报价、预约试驾、加入对比。

### 5.6 跟进话术助手

MVP 应覆盖高频销售场景：

- 首次加微欢迎语。
- 邀约试驾。
- 价格异议处理。
- 竞品对比。
- 贷款/全款方案解释。
- 置换沟通。
- 试驾后跟进。
- 订金催付。
- 交车前提醒。

话术生成需要可编辑、可复制、可发送企微，并保留发送记录。

### 5.7 报价与订单

参考“合同订单”模块，MVP 先支持轻量订单链路：

- 新建报价单。
- 生成落地价明细。
- 优惠申请与经理审批。
- 订金记录。
- 订单状态流转。
- 收款记录。
- 业绩归属。

阶段状态建议：

线索 -> 已建档 -> 已邀约 -> 已试驾 -> 已报价 -> 已付订金 -> 已签合同 -> 已交车 -> 已回访。

### 5.8 看板和报告

MVP 至少提供：

- 今日线索和首响。
- 试驾预约/到店。
- 报价和订金。
- 成交订单和交车。
- 顾问排行。
- 渠道转化。
- 库存预警。
- 日报/月报自动总结。

## 6. 智能体能力设计

### 6.1 总控智能体

负责理解用户意图并路由到具体能力：

- 打开模块。
- 查询客户/库存/订单。
- 生成推荐。
- 生成话术。
- 做数据分析。
- 创建任务或推进流程。

### 6.2 线索清洗智能体

输入：Excel、API、表单、聊天记录。

输出：

- 标准化客户字段。
- 线索有效性评分。
- 重复线索合并建议。
- 来源渠道和活动归因。
- 初始标签。

### 6.3 需求识别智能体

输入：客户备注、聊天、电话摘要、浏览行为。

输出：

- 预算。
- 用车场景。
- 车型偏好。
- 决策周期。
- 付款方式倾向。
- 关键顾虑。
- 成交概率。

### 6.4 车型推荐智能体

输入：客户需求、车型库、库存、优惠政策。

输出：

- 推荐车型排序。
- 推荐理由。
- 竞品对比。
- 试驾邀约建议。
- 备选方案。

### 6.5 报价方案智能体

输入：车型、库存、门店政策、客户预算、金融/保险规则。

输出：

- 落地价估算。
- 全款/贷款对比。
- 金融月供方案。
- 置换补贴建议。
- 可申请优惠范围。

注意：最终价格和金融承诺必须走审批或人工确认，AI 只给建议和草案。

### 6.6 跟进话术智能体

输入：客户画像、当前阶段、历史沟通、顾虑点。

输出：

- 微信话术。
- 电话提纲。
- 反驳点和证据。
- 追问问题。
- 下次跟进时间。

### 6.7 风险预警智能体

识别以下风险：

- 高意向客户超时未跟进。
- 已试驾未报价。
- 已报价长期未订。
- 金融材料未补齐。
- 订金后车辆资源风险。
- 热门库存不足。
- 滞销库存库龄过长。
- 顾问跟进质量异常。

### 6.8 经营分析智能体

自动生成日报/月报：

- 今日成交和试驾表现。
- 渠道转化变化。
- 库存结构建议。
- 顾问表现和辅导建议。
- 下周重点客户清单。

## 9. 典型流程

### 9.1 新线索首次跟进

1. 抖音直播线索进入系统。
2. AI 识别客户预算 20 万、偏新能源 SUV、两周内购车。
3. 系统合并历史咨询记录，发现客户看过同级竞品。
4. AI 推荐 2 个现车配置和 1 个备选配置。
5. 销售顾问一键生成加微话术并发送。
6. 系统自动创建“明天 10:00 邀约试驾”的待办。

### 9.2 到店试驾后报价

1. 顾问记录试驾反馈：空间满意，但价格敏感。
2. AI 总结顾虑，建议突出置换补贴和金融贴息。
3. 顾问生成全款/贷款两版报价。
4. 超出优惠阈值时进入经理审批。
5. 审批通过后生成报价单并记录发送。

### 9.3 经理晨会

1. 经理输入“分析昨天线索到试驾的掉点”。
2. AI 返回：某渠道线索多但首响慢，三名顾问有超时未跟进。
3. 系统列出待抢救客户和建议动作。
4. 经理批量分配任务并跟踪完成。

## 12. 关键指标

| 指标 | 含义 |
| --- | --- |
| 首响时长 | 线索进入到首次联系的时间 |
| 有效线索率 | 有真实购车需求的线索占比 |
| 加微率 | 可持续触达客户占比 |
| 邀约率 | 产生试驾邀约的线索占比 |

| 报价转订率 | 报价后支付订金占比 |
| 订单成交率 | 线索最终成交占比 |
| 金融渗透率 | 成交订单中使用金融方案占比 |

## 13. 产品边界和风控

- AI 可以生成报价草案，但最终成交价、金融利率、保险价格必须以门店审批和实际系统为准。
- AI 推荐应展示依据，避免黑盒推荐影响销售顾问信任。
- 客户隐私数据需要权限控制和操作日志。
- 对外发送的 AI 话术需要顾问确认，不建议完全自动发送。
- 库存和价格强依赖实时数据，生产环境必须接入 DMS/OEM/门店政策源。
- 金融保险涉及合规，不应让 AI 独立承诺审批结果。

## 14. 建议的第一版演示范围

第一版演示不需要做完整后端，重点让“汽车销售闭环”跑通：

1. 首页输入：“客户预算 20 万，想买新能源 SUV，三口之家，周末试驾。”
2. AI 返回 3 个车型推荐卡片。
3. 点击车型进入客户详情，展示需求画像和行为轨迹。
4. 一键生成微信跟进话术。
5. 创建试驾预约。
6. 生成报价单。
7. 报价进入订单列表，展示审批、订金、交车状态。
8. 报告中心展示本周线索、试驾、报价、订单、库存风险。

这个范围最贴近参考实现的结构，也最容易快速改造成可交互原型。

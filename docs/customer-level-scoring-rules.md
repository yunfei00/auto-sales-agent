# 客户等级评分规则 V1.0

> 状态：草案  
> 适用范围：汽车销售线索与未成交客户  
> 核心原则：客户等级是动态计算结果，不是永久固定标签。

## 1. 目的

客户等级用于表示客户当前的成交优先级，帮助销售顾问和销售经理判断：

- 哪些客户需要立即跟进；
- 哪些客户值得重点投入时间；
- 哪些客户需要持续培育；
- 哪些客户已经降温或基本无效。

系统应始终展示客户的最新等级，并保留等级变化历史。

## 2. 客户等级与销售阶段的区别

### 2.1 客户等级

客户等级表示客户当前的成交优先级和跟进价值：

- A+：必须优先抢单；
- A：高意向；
- B：正常培育；
- C：低意向；
- D：无效或极弱线索；
- N：信息不足，暂时无法判断。

### 2.2 销售阶段

销售阶段表示客户当前走到了哪个业务步骤：

新线索 → 已联系 → 已邀约 → 已预约试驾 → 已试驾 → 已报价 → 已付订金 → 已签合同 → 已交车

客户等级和销售阶段必须同时存在。两名客户即使都处于“已报价”阶段，也可能因为购车时间、互动情况、预算匹配度不同，而分别属于 A+ 和 C。

## 3. 评分结构

客户评分满分为 100 分，由六个加分维度和风险扣分组成。

| 评分维度 | 最高分 | 判断内容 |
| --- | ---: | --- |
| 购车时间紧迫度 | 25 | 客户计划什么时候买车 |
| 销售推进阶段 | 25 | 是否联系、邀约、试驾、报价 |
| 需求明确程度 | 15 | 预算、车型、配置、用途是否明确 |
| 交易可行性 | 15 | 预算是否匹配，付款、贷款、置换是否可行 |
| 互动活跃程度 | 10 | 最近是否回复、咨询、到店或主动联系 |
| 决策准备程度 | 10 | 是否明确决策人、竞品范围和主要顾虑 |
| 风险扣分 | 最多扣 20 | 失联、预算不符、竞品锁定、虚假信息等 |

计算公式：

```text
客户评分 = 六项基础得分 - 风险扣分
```

评分最低为 0 分，最高为 100 分。

## 4. 详细评分规则

### 4.1 购车时间紧迫度：最高 25 分

| 预计购车时间 | 得分 |
| --- | ---: |
| 7 天以内 | 25 |
| 8—15 天 | 20 |
| 16—30 天 | 15 |
| 31—60 天 | 10 |
| 61—90 天 | 5 |
| 90 天以后 | 2 |
| 未知 | 0 |

客户明确表示“今天能谈好就定”“本周购买”等信息时，可以按照 7 天以内计算。客户购车时间推迟后，应立即重新计算评分。

### 4.2 销售推进阶段：最高 25 分

| 当前阶段 | 得分 |
| --- | ---: |
| 新线索，尚未联系 | 0 |
| 已完成首次联系 | 4 |
| 已确认基本需求 | 7 |
| 已邀约到店或试驾 | 10 |
| 已预约试驾 | 13 |
| 已完成试驾 | 17 |
| 已生成正式报价 | 21 |
| 正在确认优惠、金融或订金 | 25 |

客户支付订金、签订合同后，不再继续使用普通意向等级，应进入已成交、待交付、已交付或售后运营状态。

### 4.3 需求明确程度：最高 15 分

按照以下信息完整程度累计评分：

| 信息 | 得分 |
| --- | ---: |
| 预算范围明确 | 3 |
| 意向车型或车型类型明确 | 3 |
| 能源类型明确 | 2 |
| 主要用途明确 | 2 |
| 配置、颜色、座位等偏好明确 | 2 |
| 购车城市或上牌地区明确 | 1 |
| 主要关注点明确 | 2 |

主要关注点包括价格、续航、油耗、空间、智能驾驶、安全、品牌、保值率和交付周期。

### 4.4 交易可行性：最高 15 分

| 情况 | 得分 |
| --- | ---: |
| 客户预算能够覆盖目标车型落地价 | 5 |
| 客户预算略有差距，但可通过优惠或金融解决 | 3 |
| 已明确全款或贷款方式 | 3 |
| 贷款资质和月供范围基本可行 | 2 |
| 置换车辆信息明确 | 2 |
| 购车资格、上牌条件无明显问题 | 3 |

本项最高累计 15 分。预算差距过大、贷款资质存在明显问题、置换价格预期严重不合理或上牌资格存在障碍时，应降低本项得分。

### 4.5 互动活跃程度：最高 10 分

| 最近一次有效互动 | 得分 |
| --- | ---: |
| 24 小时以内 | 10 |
| 2—3 天以内 | 8 |
| 4—7 天以内 | 5 |
| 8—14 天以内 | 2 |
| 超过 14 天 | 0 |

以下行为可以视为高质量互动：

- 客户主动发起咨询；
- 主动询问优惠或现车；
- 主动确认试驾时间；
- 主动询问贷款、置换或交付；
- 到店看车或完成试驾；
- 回复销售提出的关键问题。

群发消息已读、系统通知送达等行为不应视为有效互动。

### 4.6 决策准备程度：最高 10 分

| 情况 | 得分 |
| --- | ---: |
| 已明确最终决策人 | 2 |
| 家庭或共同决策人意见基本一致 | 2 |
| 已明确主要竞品范围 | 2 |
| 已明确客户最重要的成交条件 | 2 |
| 客户愿意接受下一步成交动作 | 2 |

下一步成交动作包括到店、试驾、获取正式报价、提交贷款预审、评估置换车辆、申请优惠和支付订金。

## 5. 风险扣分规则

风险项采用扣分制，累计最多扣 20 分。

| 风险情况 | 扣分 |
| --- | ---: |
| 连续多次联系无回应 | -5 |
| 报价后超过 7 天无反馈 | -5 |
| 试驾后超过 5 天无有效互动 | -4 |
| 预算与目标车型差距明显 | -5 |
| 客户明确表示购车计划推迟 | -5 |
| 已基本锁定竞品 | -6 |
| 联系方式无效或身份信息明显异常 | -10 |
| 客户明确表示不再购买 | -20 |
| 重复线索或非真实购车咨询 | -20 |

系统必须保存每一个风险原因，不能只保存最终扣分值。

## 6. 客户等级映射

| 客户等级 | 分数范围 | 业务含义 |
| --- | ---: | --- |
| A+ 必抢客户 | 85—100 | 短期内极可能成交，需要最高优先级跟进 |
| A 高意向客户 | 70—84 | 真实需求明确，近期有较高成交机会 |
| B 培育客户 | 50—69 | 有真实需求，但条件、时间或意愿尚未成熟 |
| C 低意向客户 | 25—49 | 需求模糊、购车时间较远或互动较弱 |
| D 无效或弱线索 | 0—24 | 暂时不值得销售重点投入 |
| N 待判断客户 | 暂不评分 | 有效信息不足，尚未完成首次需求识别 |

## 7. N 级客户判断条件

满足以下任意情况时，可以先标记为 N 级：

- 新线索尚未完成首次联系；
- 只有姓名或手机号，没有有效需求信息；
- 无法判断预算、车型和购车时间；
- 系统刚完成线索导入，尚未完成数据清洗；
- 沟通内容不足以支持可靠评分。

N 级不是低意向，而是“暂时不知道”。系统应自动生成“尽快完成首次联系和需求确认”的待办任务。

## 8. 动态重新计算机制

以下信息发生变化时，应立即重新计算客户评分和等级：

- 预算更新；
- 意向车型更新；
- 购车时间更新；
- 付款方式或置换意向更新；
- 新增聊天、电话或到店记录；
- 客户主动咨询；
- 销售阶段变化；
- 新增或完成试驾；
- 创建或更新报价；
- 申请优惠；
- 客户支付订金；
- 竞品信息或风险信息更新。

系统还应通过定时任务处理时间流逝导致的变化：

- 关键信息更新后立即计算；
- 每天凌晨对全部未成交客户重新计算一次；
- 对高意向客户每小时检查超时和降级风险。

页面不应在每次打开时依赖大模型临时重新判断，而应直接读取数据库中保存的最新评分结果。

## 9. 等级变更规则

客户等级允许自动升级和自动降级。

### 9.1 常见升级情况

- 客户购车时间由三个月后变为本月；
- 客户主动预约或完成试驾；
- 客户接受正式报价；
- 客户询问贷款、置换或订金；
- 客户明确最终决策条件；
- 客户主动要求申请优惠。

### 9.2 常见降级情况

- 客户购车计划推迟；
- 多日未回复；
- 试驾后长期无反馈；
- 报价后失联；
- 预算无法覆盖目标车型；
- 竞品已经给出更有吸引力的方案；
- 客户明确暂不购车；
- 联系信息无效。

## 10. 等级变化历史

每次等级发生变化时，系统需要记录：

- 客户 ID；
- 原等级和新等级；
- 原评分和新评分；
- 触发事件；
- 六项评分明细；
- 风险扣分明细；
- 升降级原因；
- 计算时间；
- 是否由人工调整；
- 操作人员。

示例：

```text
客户：王先生
原等级：B（64 分）
新等级：A（77 分）
原因：完成试驾，购车时间确认在 15 天内
时间：2026-07-11 16:30
```

## 11. 人工调整规则

销售主管可以人工调整客户等级，但必须填写原因，并设置有效期，例如 24 小时、3 天或 7 天。有效期结束后，系统重新恢复自动计算。

页面应同时展示：

- 系统建议等级；
- 当前执行等级；
- 人工调整原因；
- 人工调整到期时间。

人工调整不能永久覆盖自动评分。

## 12. 页面展示建议

客户列表至少展示：

- 客户姓名；
- 当前销售阶段；
- 当前客户等级；
- 客户评分；
- 最近互动时间；
- 预计购车时间；
- 下一步动作；
- 负责人。

客户详情页应展示：

- 当前等级和评分；
- 等级更新时间；
- 六项评分明细；
- 风险扣分项；
- 本次判断原因；
- 历史等级变化；
- AI 建议的下一步动作。

## 13. V1.0 实施原则

第一版采用固定规则评分，确保：

- 结果可解释；
- 销售能够理解；
- 经理能够审核；
- 规则能够快速调整；
- 出错时能够定位原因。

AI 负责从聊天、电话摘要和客户备注中提取预算、购车时间、车型偏好、顾虑和竞品等结构化信息；最终等级由确定性的评分规则计算，避免大模型每次直接判断等级造成结果不稳定。

后续积累真实成交数据后，再根据成交结果、成交周期、流失原因、渠道转化率、车型转化率和销售跟进效果持续校准评分权重。
